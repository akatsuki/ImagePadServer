# おすそ分け機能メモ

VRChat を VR ゴーグル本体の Android 版だけで遊んでいる人は、ImagePadServer を動かす PC を持っていないことがあります。そこで、PC を持っているユーザーが自分の ImagePadServer を一時的に開放し、ほかのユーザーも画像を投稿できるようにする機能を「おすそ分け」と呼びます。

## 目的

- PC を持たない Quest / Android VRChat ユーザーでも、ほかの人の ImagePadServer 経由で画像を公開できるようにする。
- サーバーを開く人は、公開範囲と終了タイミングを明確に管理できるようにする。
- 参加者には管理画面を見せず、画像投稿だけを許可する。

## 想定フロー

1. サーバー主が管理画面で `おすそ分けを開始` を押す。
2. ImagePadServer が Cloudflare Tunnel の公開 URL とは別に、投稿専用 URL を発行する。
3. サーバー主が VRChat 内で投稿専用 URL または QR コードを共有する。
4. 参加者はスマホや VR ゴーグルのブラウザから画像を選んで投稿する。
5. 投稿された画像はサーバー主の画面で確認され、承認後に現在の公開画像として反映される。
6. サーバー主が `おすそ分けを終了` を押すと、投稿専用 URL は無効になる。

## 権限設計

- おすそ分けは既定でオフ。
- 開始したセッションには推測しにくい共有トークンを付ける。
- 参加者が使えるのは投稿専用ページと投稿 API だけ。
- 参加者は管理 API、履歴、お気に入り、動画 URL 取り込み、設定変更へアクセスできない。
- Cloudflare Tunnel 経由の管理画面アクセス拒否は維持する。

## 投稿の扱い

- 初期実装は画像のみを対象にする。
- 投稿はすぐ公開せず、サーバー主の承認待ちキューに入れる。
- サーバー主は承認、却下、投稿者名の確認、キューの全削除ができる。
- 投稿者名は任意入力にする。VRChat アカウント連携はしない。
- 同時投稿が重なった場合は、受信順にキューへ積む。

## 制限

- 1 セッションあたりの投稿数上限を設ける。
- 1 ファイルあたりのサイズ上限は通常アップロードと同じか、より小さくする。
- 連続投稿を抑えるため、IP 単位の簡易レート制限を入れる。
- セッションには有効期限を設ける。
- 終了時に未承認キューを破棄する。

## 実装候補

- `internal/server/server.go`
  - おすそ分け開始/終了 API、投稿専用 API、承認/却下 API を追加する。
- `internal/server/ui.go`
  - 管理画面におすそ分けタブまたは小さな操作パネルを追加する。
  - 投稿専用ページを追加する。
- `internal/settings/settings.go`
  - 既定値や上限など、永続化が必要な設定だけを保存する。
- `internal/library/store.go`
  - 承認待ちキューを現在公開中メディアと分離して扱う。

## 未決定事項

- 投稿専用 URL を Cloudflare Tunnel の同一ドメイン配下に置くか、別経路にするか。
- 承認待ちキューをメモリだけにするか、セッション中だけディスクへ置くか。
- 動画投稿まで広げるか。初期実装では対象外にするのが安全。
- ワールド内で共有しやすい短い合言葉や短縮コードを用意するか。
