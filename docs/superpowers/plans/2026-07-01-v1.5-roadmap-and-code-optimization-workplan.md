# v1.5 Roadmap and Code Optimization Workplan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Extract the v1.5 roadmap into version-independent work steps, because the local version labels have drifted from the original `v1.4.6 -> v1.5.0` roadmap.

**Architecture:** Treat each roadmap entry as a shippable workstream instead of binding it to the stale intermediate version numbers. Keep behavior fixes before refactors, then do the code-optimization splits that reduce ownership drift, and add image-format feature work after the high-risk routing and toolchain boundaries are clearer. v1.6 product features stay out of this plan, but the code split work needed to make v1.6 safer is included as an explicit optimization track.

**Tech Stack:** Go, net/http, embedded server UI, image processing, FFmpeg-backed media tooling, Windows release resources.
